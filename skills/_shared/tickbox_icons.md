# Shared rule: status tickbox icons

Work-summary and status lines use this icon set. Pick the one icon that matches each line's
real outcome; keep each line to about 10 words. Referenced by the skills that emit an
end-of-turn or status report.

| Icon | Meaning |
|---|---|
| 🟢 | good / on track |
| 🟡 | minor caution |
| 🟠 | notable concern / at risk |
| 🔴 | blocked / problem |
| 🔵 | informational note |
| ✅ | done / pass |
| ❌ | not done / failed |
| ⚠️ | warning / caveat |
| ⚖️ | judgment call / trade-off |
| ⏳ | pending / in progress |

Match the icon to the actual result of each line - a failed or skipped step is ❌, not ✅.
These belong to internal work summaries and status reports only; developer-facing prose
(Jira comments, bug reports) uses none.
